**Pane is a browser with a soul.**

An open-source browser where the assistant is built in — not bolted on. It learns from how you work, remembers what matters, and becomes whatever role you need: chief of staff, job search partner, research buddy, or something you have not named yet.

Your work lives in tabs: docs, dashboards, tickets, research, meetings, internal tools. A normal AI assistant lives somewhere else and only sees what you paste in. Pane starts from the work itself, because the agent *is* the browser — and over time, it becomes *yours*.

**Open source. Local-first. No Pane account. Designed to grow with you.**

## The idea

Separating your work from the assistant that helps with your work is a bug.

External agents — Hermes, OpenClaw — are full personal agents: memory, skills, scheduled work, files, terminal, reach. But they reach your browser from the outside. They see a snapshot, not your situation.

AI browsers — Atlas, Comet, Dia — put AI in the browser, but they are still mostly **chatbots with page context**. Ask about the tab. Summarize a doc. They are not built to become a persistent, self-improving agent that runs your work over time.

Pane takes a third path: **a Hermes-class agent that is the browser.** Same session, same logins, same tabs — plus files and terminal, plus a roadmap toward memory, capture, and proactive work. Local and open source.

> Pane is a fork of [BrowserOS](https://github.com/browseros-ai/BrowserOS). BrowserOS is a great project — we recommend it if you want their product. We forked to pursue a more personal, local-first agent browser with no Pane-operated servers.

Other directions: decisions tracked across shopping tabs over several days; webmail triage using your project context; localhost that already knows your ticket and branch because the browser, workspace, and terminal are one loop.

This is not "summarize this page." It is a browser that understands your work and reshapes itself around you — locally, with your consent, on your machine.

## What works today

Pane is early. The foundation is real; the full vision is still being built.

<Columns cols={3}>
  <Card title="Chat & Agent" icon="message">
    Ask about the page you are on. Describe a task and Pane will navigate, click, type, and extract — in your real browser session, with your real logins.
  </Card>
  <Card title="Files & terminal" icon="folder-open">
    Grant Pane a workspace folder. It can read, write, search, and run commands — so one task can span the web and your machine.
  </Card>
  <Card title="Your editor's browser" icon="terminal">
    Pane is an MCP server. Claude Code, Cursor, and Gemini CLI can drive your actual browser — localhost, authenticated apps, console output — not a headless fake session.
  </Card>
</Columns>

Pane also supports scheduled tasks, custom MCP integrations, Chrome extension compatibility, and bring-your-own-model (API keys, OAuth subscriptions, or local models like Ollama). See the feature docs in the sidebar.

## Who Pane is for

**Developers first.** The sharpest proof of the thesis: point your coding agent at Pane, reproduce a UI bug in your real session, read the console, fix the code, re-verify — one loop, no context switching. That is the wedge.

**Then everyone who lives in tabs.** Researchers threading pages toward a question. Knowledge workers drafting from dashboards. Anyone tired of copying URLs into ChatGPT. Anyone who wants a personal agent without running a VPS or a daemon.

**And anyone who will not send their browsing to a vendor cloud.** Pane is local-first and open source by design.

## Get started

<Steps>
  <Step title="Download Pane">
    Get the latest release from [GitHub releases](https://github.com/abhishek-verma/Pane/releases) for macOS, Windows, or Linux.
  </Step>
  <Step title="Import from Chrome">
    Go to `chrome://settings/importData` to bring over bookmarks, passwords, and extensions.
  </Step>
  <Step title="Connect your model">
    Pane does not pick a vendor for you. Bring an API key, sign in with ChatGPT Pro / GitHub Copilot / Qwen via OAuth, or run a local model.

    <Card title="Set up your LLM provider" icon="key" href="/features/bring-your-own-llm">
      Configure Claude, OpenAI, Gemini, or Ollama
    </Card>
  </Step>
  <Step title="Use it">
    Press **Assistant** in the toolbar on any page. Or open **Pane as MCP** in settings and connect Claude Code in one line.
  </Step>
</Steps>

## Which model?

<Tabs>
  <Tab title="Chat">
    Any model works, including local ones. Ollama and LM Studio are fine for page Q&A, summaries, and extraction.
  </Tab>
  <Tab title="Agent">
    Agent mode needs stronger reasoning — Claude Sonnet or Opus recommended. Local models are not reliable enough for multi-step browser automation yet.

    <Warning>
    For Agent mode, use a capable cloud model. Local LLMs work well for Chat, not yet for clicking and navigating.
    </Warning>
  </Tab>
</Tabs>

## Why Pane feels different

- **The agent is the browser**, not a guest in it.
- **Your real session** — logins, extensions, localhost, authenticated apps.
- **Web + machine in one loop** — browser, files, terminal together.
- **Gets personal over time** — memory, skills, and capture are the long game.
- **Yours** — open source, local-first, no required Pane account.
